# v5 — Transcript 結構化抽取（支援非共識洞察與估值假設）

## 目標
用你們的內部 transcript（earningscall.biz）把文章 2/3 變成真正「機構級」：
- 法說後 1 小時內就能拿到 transcript → 08:00 不需要中午補強
- 將 transcript 轉為結構化訊號，支援：
  - 非共識觀點
  - 估值假設（guidance、margin、capex、需求）
  - 風險敘事（競爭、供應鏈、監管）

---

## 範圍（Scope）
### 會做
- `ingest/transcript_client.py`：串內部 API（以你們實際接口為準）
- Transcript 抽取（Extraction v1）
  - 只抽取結構化摘要，不要存整篇全文到文章（版權/可讀性）
  - 結構欄位：
    - guidance/outlook（數字如有，必須可引用來源；無則不寫數字）
    - risks
    - new products/strategy
    - tone（偏保守/樂觀/防禦）
    - Q&A 中最關鍵的 3 個問題（摘要）
- 將抽取結果注入：
  - 文章 2：管理層訊號（Management Signals）章節
  - 文章 3：主題共識/分歧（Theme Consensus vs Divergence）

### 不做
- SEC/XBRL 交叉驗證（v6）
- 13F（v7）

---

## 交付物（Deliverables）
- `llm/extract_transcript_json.py`
- `schemas/transcript_extract.schema.json`（或 pydantic model）
- 文章 2 Evidence Pack 增加 `management_signals`
- 文章 3 Evidence Pack 增加 `theme_voice_of_management`

---

## 實作要點
- transcript 很長，建議兩段式：
  1) chunk 摘要 → JSON
  2) 合併 JSON → 最終 signals（避免 token 爆）
- 任何引用 transcript 的內容：
  - 以「改寫摘要」為主
  - 只在必要時引用短句（且控制字數）

---

## 驗收標準（Definition of Done）
- [ ] 文章 2 每日可取得（或判斷無 transcript）並產出管理層訊號段落
- [ ] 文章 2 的估值假設有引用 management signals（至少 1–2 個連動）
- [ ] 文章 3 能引用多家公司管理層觀點形成主題敘事（若資料足夠）
- [ ] 仍可在 08:00 穩定跑完（無中午補強）
